# Лекция №2

### Архитектуры процессоров и основные проблемы

#### Многообразие архитектур

* Основные типы: x86, ARM, AVR
* Необходимость стандартизации при сохранении обратной совместимости
* Эволюция процессоров x86:
  * 16-битные (8086-80286): адресное пространство до 64 КБ
  * 32-битные (80386 и далее): адресное пространство до 4 ГБ
  * 64-битные (AMD64/EM64T/x86-64): адресное пространство до \~18.4 эксабайт

#### Особенности регистров

* Размер регистров кратен размеру ячейки памяти (8, 16, 32, 64 бита)
* Два типа адресации: байтовая и словесная
* Байтовая адресация предпочтительнее для работы со строками и ASCII-символами
* Названия регистров отражают их назначение:
  * EAX/AX/AH/AL (аккумулятор)
  * EBX/BX/BH/BL (базовый регистр)
  * ECX/CX/CH/CL (счетчик)
  * EDX/DX/DH/DL (регистр данных)
  * ESI/SI (индекс источника)
  * EDI/DI (индекс приемника)
  * ESP/SP (указатель стека)
  * EBP/BP (указатель базы кадра стека)

### Режимы работы процессора

#### Режимы процессора

* Режим реальных адресов
* Защищенный режим
* 64-разрядный режим

#### Защищенный режим

* Защита данных операционной системы от прикладных программ
* Защита данных прикладных программ друг от друга
* Использование уровней привилегий:
  * Уровень 0: операционная система
  * Уровни 1-2: промежуточные уровни
  * Уровень 3: пользовательские приложения

#### Адресация в защищенном режиме

* **Виртуальная память** с трансляцией виртуальных адресов в физические
* **Логический адрес**: селектор дескриптора (16 бит) + смещение в сегменте (32 бита)
* **Селектор дескриптора** содержит:
  * Уровень привилегий
  * Номер таблицы
  * Индекс в таблице
* Физический адрес вычисляется как: адрес начала сегмента + смещение в сегменте

#### Страничная организация памяти

* Виртуальный адрес разделяется на три компонента:
  * Индекс в каталоге страниц
  * Индекс в таблице страниц
  * Смещение в странице
* Иерархическая структура: каталог страниц → таблицы страниц → страницы памяти
* Процесс трансляции: логический адрес → виртуальный адрес → физический адрес

### Прерывания и многозадачность

#### Прерывания

* Сигнал процессору для прерывания текущей программы
* Передача управления специальной функции-обработчику прерывания

#### Многозадачность

* Организация работы нескольких процессов
* Каждый процесс содержит несколько задач
* Поддержка многопроцессорности в современных системах

### Кэш-память процессора

#### Принципы организации кэша

* **Кэш-строка**: оптимальный формат хранения (данные + адрес)
* Размер кэш-памяти кратен степени числа 2
* Полный размер кэша = размер данных + размер служебных данных
* Пример: кэш на 128 байт с 4 строками по 32 байта имеет полный размер 144 байта

#### Типы кэшей

1. **Кэш прямого отображения**: простая, но неэффективная организация
2. **Наборно-ассоциативный кэш**: оптимальный баланс скорости и эффективности
3. **Полностью ассоциативный кэш**: наиболее эффективный, но медленный

#### Стратегии замещения данных

* **LRU** (least recently used): замещаются ресурсы, к которым дольше всего не обращались
* **MFU** (most frequently used): замещаются последнее использованные данные
* **LFU** (least frequently used): замещаются ресурсы, которые использовались реже всего

#### Многоуровневый кэш

* **L1-кэш**: самый быстрый, принадлежит конкретному ядру, содержит данные и инструкции
* **L2-кэш**: медленнее L1, также привязан к ядру, содержит данные и инструкции
* **L3-кэш**: самый большой и медленный, общий для всех ядер

#### Архитектуры кэшей

* **Инклюзивная (включающая)**: L2 дублирует содержимое L1
* **Эксклюзивная (исключающая)**: L2 не дублирует содержимое L1, оптимизирует использование памяти

### Базовые команды ассемблера

#### Команды пересылки данных

**Команда MOV**

* Пересылает данные между регистром и памятью или между двумя регистрами
* Синтаксис: MOV приемник, источник
* Не поддерживает прямую пересылку между ячейками памяти
*   Пример для присвоения одной переменной значения другой:

    ```assembly
    MOV AX, A
    MOV B, AX
    ```

**Команда XCHG**

* Меняет местами содержимое двух операндов
* Не работает с непосредственно заданными значениями
*   Для обмена значениями двух переменных в памяти:

    ```assembly
    MOV EAX, VAL1
    XCHG EAX, VAL2
    MOV VAL1, EAX
    ```

#### Арифметические команды

**Команды сложения**

* **ADD**: простое сложение 8-, 16-, 32-битовых операндов
* **XADD**: сначала обмен данными, затем сложение
*   **ADC**: сложение с учетом флага переноса CF

    ```assembly
    ; Сложение 64-битовых чисел
    ADD EAX, ECX  ; сложение младших 32 бит
    ADC EBX, EDX  ; сложение старших 32 бит с учетом переноса
    ```

**Команды вычитания**

* **SUB**: простое вычитание
*   **SBB**: вычитание с учетом флага переноса

    ```assembly
    ; Вычитание 64-битовых чисел
    SUB EAX, ECX  ; вычитание младших 32 бит
    SBB EBX, EDX  ; вычитание старших 32 бит с учетом заема
    ```

**Инкремент и декремент**

* **INC**: добавляет 1, не влияет на флаг переноса CF
* **DEC**: вычитает 1, не влияет на флаг переноса CF
* **NEG**: обращает знак операнда (вычитает значение из нуля)

#### Команды умножения и деления

**Умножение**

* **MUL**: умножение беззнаковых чисел
* **IMUL**: умножение чисел со знаком
* Операнды: один указывается явно, второй берется из AL/AX/EAX
* Результат имеет двойной размер:
  * Для байтов: в регистрах AH (старший) и AL (младший)
  * Для слов: в регистрах DX (старшее) и AX (младшее)
  * Для двойных слов: в регистрах EDX и EAX

**Деление**

* **DIV**: деление беззнаковых чисел
* **IDIV**: деление чисел со знаком
* Делимое имеет двойной размер и берется из:
  * AH и AL (при 8-битовом делителе)
  * DX и AX (при 16-битовом делителе)
  * EDX и EAX (при 32-битовом делителе)
* Результат:
  * Частное помещается в AL/AX/EAX
  * Остаток помещается в AH/DX/EDX

### Флаги процессора при арифметических операциях

#### Флаги при сложении/вычитании

* **CF** (Carry Flag): устанавливается при переносе/заеме
* **PF** (Parity Flag): устанавливается при четном количестве единиц в младших 8 битах
* **ZF** (Zero Flag): устанавливается, если результат равен нулю
* **SF** (Sign Flag): устанавливается, если результат отрицательный
* **OF** (Overflow Flag): устанавливается при переполнении:
  * При сложении чисел одного знака, если результат меняет знак
  * При вычитании чисел разного знака, если результат превышает диапазон

#### Особенности флагов при умножении

* Для **MUL**: флаги CF и OF равны 0, если старшая половина произведения равна 0
* Для **IMUL**: флаги CF и OF равны 0, если старшая половина является расширением знака младшей половины
