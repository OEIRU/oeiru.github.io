# Лекция №1

### Введение и организационные моменты

#### Зачем изучать ассемблер?

* Понимание работы программ на уровне "железа"
* Возможность определить, не является ли проблема следствием ошибки компилятора
* Необходимость для задач обратного инжиниринга

#### Структура курса

* Лекции
* Лабораторные работы
* Расчетно-графическое задание (РГЗ) для групп ПМ
* Форма отчетности: зачет (возможен автомат, при своевременном выполнении всех работ)

#### Правила аттестации

Для групп ПМ и ПМИ установлены различные системы оценки, включающие:

* Баллы за лабораторные работы
* Баллы за РГЗ/самостоятельные работы
* Минимальные пороги для получения автоматического зачета

#### Лабораторные работы

**Содержание отчета:**

* Задание в соответствии с вариантом
* Используемые программные средства
* Текст программы с комментариями
* Набор тестов для проверки

**Процесс сдачи:**

* Демонстрация отчета в печатном виде
* Демонстрация работы программы
* Защита с ответами на контрольные вопросы

### Базовые понятия архитектуры процессоров

#### Архитектура vs Ассемблер

* **Язык ассемблера** – язык программирования, состоящий из команд процессора в символьном представлении
* **Ассемблер** – компилятор, преобразующий язык ассемблера в машинный код

#### Архитектура процессоров

* Представляет собой совместимость с определенным набором команд
* Включает структуру команд, систему адресации, набор регистров и механизм исполнения
* **Обратная совместимость** – программы для старых моделей должны работать на новых

#### Эволюция процессоров

* 8086, 80186, 80286 – 16-разрядные процессоры
* 80386, 80486 – 32-разрядные процессоры
* Core i7 и современные – 64-разрядные процессоры

### Регистры процессора

#### Регистры общего назначения

Эволюция регистров:

* 16-разрядные: AX, BX, CX, DX
* 32-разрядные: EAX, EBX, ECX, EDX
* 64-разрядные: RAX, RBX, RCX, RDX

#### Указатели и индексы

* BP/EBP – указатель базы
* SP/ESP – указатель стека
* IP/EIP – указатель команд (недоступен программисту напрямую)
* SI/ESI – индекс источника
* DI/EDI – индекс приемника

#### Сегментные регистры

* CS – сегмент кода
* DS – сегмент данных
* SS – сегмент стека
* ES, GS, FS – дополнительные сегменты данных

#### Регистр состояния FLAGS

Содержит три группы флагов:

* Флаги состояния – отражают результат арифметических/логических операций
* Флаг направления – определяет направление обработки цепочек
* Системные флаги – управляют вводом/выводом, прерываниями, отладкой

### Компиляторы ассемблера и синтаксис

#### Синтаксические различия

Два основных формата:

* **INTEL**: `mov ax, 3` (сначала приемник, потом источник)
* **AT\&T**: `movw %ax, %bx` (размер операнда указывается суффиксом: b, w, l)

### Модель памяти и сегменты программы

#### Сегменты программы

* **Сегмент кода** – содержит выполняемые команды
* **Сегмент данных** – содержит инициализированные переменные
* **Сегмент стека** – используется для хранения локальных переменных и адресов возврата

#### Формирование физического адреса

* **Логический адрес**: сегмент:смещение (16 бит:16 бит)
* **Физический адрес**: адрес\_сегмента << 4 + смещение
* Максимальный размер сегмента – 64 КБ
* Минимальный размер сегмента – 16 байт

#### Создание сегментов в коде

Два основных способа:

1.  Явное определение сегментов:

    ```assembly
    code SEGMENT
    ...
    code ends
    ```
2.  Использование директивы .model и специальных секций:

    ```assembly
    .model flat
    .code
    .data
    .stack
    ```

#### Модели памяти

* **flat** – все в одном сегменте, 32-битные сегменты (до 4 ГБ)
* **tiny** – аналог flat для DOS (сегменты до 64 КБ)
* **small** – по одному сегменту на код, данные и стек

### Процесс компиляции

#### Компиляция в 2 прохода

1. **Первый проход**:
   * Обработка кодов операций
   * Обработка директив
   * Сборка символических имен
2. **Второй проход**:
   * Формирование объектных модулей (.obj для Windows, .o для Linux)
   * Создание таблицы точек входа
   * Формирование таблицы внешних ссылок
   * Генерация машинных команд и констант
   * Создание словаря перераспределения

### Соглашения о вызовах (Calling Conventions)

#### Основные соглашения

1. **cdecl** (C declaration):
   * Аргументы передаются через стек справа налево
   * Аргументы размером меньше 4 байт расширяются до 4 байт
   * Используется в языке Си
2. **stdcall**:
   * Аргументы передаются через стек справа налево
   * Используется в WinAPI
3. **fastcall**:
   * Первые два параметра передаются в регистрах ecx и edx
   * Остальные параметры передаются через стек справа налево

### Ввод-вывод данных в программе на ассемблере

#### Основные функции ввода-вывода

1. **Получение дескрипторов**:
   * Функция `GetStdHandle`
   * Параметры: -10 (ввод), -11 (вывод), -12 (ошибки)
   * Результат возвращается в регистре EAX
2. **Вывод текста в консоль**:
   * Функция `WriteConsoleA` (5 параметров)
   * Требуется зарезервировать буфер с текстом
   * Получить дескриптор вывода
   * Определить длину буфера
3. **Вспомогательные функции**:
   * `CharToOemA` – перекодировка строки
   * `lstrlenA` – определение длины строки
4. **Форматированный вывод**:
   * Функция `wsprintfA` – форматирование данных в строку
   * Переменное число параметров
5. **Ввод с клавиатуры**:
   * Функция `ReadConsoleA` (5 параметров)
   * Позволяет считать данные во входной буфер
